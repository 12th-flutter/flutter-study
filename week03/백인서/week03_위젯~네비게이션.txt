1. 위젯
- 정의: 앱에서 위젯은 특정한 기능을 담당한 부품, 모든 UI 요소
- 구현: 앱에서 위젯은 클래스로 구현, 이를 상속받는 다양한 위젯이 있음
ex)
text 클래스: 글자를 표시 위젯
button 클래스: 버튼 표시 위젯
- 구분: 앱의 상태가 가변 가능한지 유무에 따라 구분
1). stateless: 정적인 위젯 ex) text, image
2). statefull: 동적인 위젯 ex) 계산기 view

2. 상태 관리
- 상태: 앱의 정보나 데이터를 나타내는 것, 어떤 시점에서 앱이 “어떻게 보이는지”와 “어떻게 동작하는 지”를 결정
    
    ex) 체크박스의 체크여부, 텍스트 입력 필드의 내용, 리스트의 아이탬 등
    
- 상태 관리 기법
    - **Local State**: 작은 위젯에서 상태를 관리할 때 유용
    - **Lifting State Up**: 상태를 상위 위젯으로 이동하여 여러 위젯이 상태를 공유할 수 있도록 함
    - **Provider, Riverpod**: 상태 객체를 효과적으로 제공하고 관리하는 도구
    - **Redux, BLoC, MobX**: 큰 앱에서 복잡한 상태 관리에 사용되는 패턴 및 라이브러리

3. 네비게이션
- 앱이 제공하는 기능이나 메뉴별로 화면을 분리해서 개발하는 방법
- 이유:  기능이 많으면 화면을 구성하기 어렵고 로딩이 느려지기도 하기 때문에 네비게이션을 구현
